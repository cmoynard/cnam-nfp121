Ce document décrit le déroulement chronologique du cours de l’UE Programmation Avancée (NFP121). Le site de l’UE parisienne correspondante est http://jfod.cnam.fr/NFP121/

Les séances sont découpées en parties notées CM, TD ou TP. CM correspond à un cours, ou une partie d’un cours (dans ce dernier cas, les numéros des transparents correspondant sont précisés). Les TD correspondent à des exercices qui seront réalisés sur papier (en salle de cours). Les TP seront à réaliser par les auditeurs, chez eux.

Pour tous les sujets marqués TD ou TP, une solution est proposée.

Remarque : Pour les fichiers Java fournis, deux archives sont proposées, par exemple TP01.jar et TP01-packages.jar. La seconde propose des classes dans des paquetages, à préférer si vous utilisez un IDE comme Eclipse. La première n’utilise pas les paquetages, à préférer si vous utilisez java en ligne de commande.

Divers :

  1. Le code à utliser pour les questionnaires papier AMC1
  2. Le programme prévisionnel de l’UE2

Presentation 1 : Présentation du module

Le cours commence par une courte présentation3 du module.

Les documents nécessaires pour le cours et les TP seront mis en ligne sur ce site. Une solution des TP sera également proposée.

CM 1 : Abstraction et modularité : classes

Le cours4 présente la classe comme un moyen d’abstraction et de modularité. L’objectif est de savoir faire en objet ce que vous savez déjà faire en programmation impérative.

TD 1 : Algorithmes simples en Java

Le TD 15 propose d’écrire quelques programmes Java pour comprendre et savoir utiliser les structures de contrôle du langage.

Solution : Voici un corrigé du TD 16 possible et les fichiers associés7.

TD 2 : Développer d’autres classes

Le TD 28 propose de spécifier et programmer deux nouvelles classes. L’objectif est d’identifier de nouveaux concepts.

Solution : Voici un corrigé du TD 29 possible.

Séance 2 (15/12/2023, 09:00)

  • CNAM : 15/12/2023, 09:00 à 12:30, Cregut Xavier

TD 3 : Spécification et implantation

Le TD 310 concerne la spécification et l’implantation d’une classe. Il permet de démontrer l’intérêt de déclarer les attributs privés et explique le principe de l’accès uniforme et la règle de protection en écriture des attributs.

Solution : Voici un corrigé du TD 311 possible et les fichiers associés :

  • la solution des exercices 1 à 512, donc le point cartésien et les deux programmes de test ;
  • la version polaire du point et le premier programme de test13 ;
  • la version mixte du point et le second programme de test14.
  • la La classe Point avec les fabriques statiques pour créer des points à partir des coordonnées cartésiennes ou des coordonnées polaires15

Remarque : Cette solution est partielle puisque les modifications pour les exercices 4, 5 et 6 ne sont pas données.

TP 1 : Points et segments

Le TP 116 propose d’utiliser les principaux outils du JDK (javac, java et javadoc) en ligne de commande, de vérifier certains aspects du langage qui ont été présentés en cours, de compléter une classe Segment et d’écrire un programme de test.

Ce TP peut aussi être fait avec https://repl.it/ en utilisant la console qui se trouve à droite de l’éditeur pour taper les commandes. Il est conseillé de s’inscrire de manière à pouvoir conserver ses fichiers d’une connexion à l’autre. Dans la console, vous pouvez aussi utiliser les commandes de subversion...

Vous partirez des classes fournies17.

TP 2 : Tester la classe Point avec JUnit

Le TP 218 propose de tester la classe Point en utilisant le framework JUnit.

Dans Eclipse, il suffit de faire un clic droit sur le projet, Build Path, Add library, JUnit, choisir la version 4.

Voici les fichiers fournis19.

TU 1 : Introduction à Eclipse

Le TU 120 montre comment utiliser Eclipse pour un projet Java. Il montre en particulier ses possibilités concernant le « refactoring » qui sera réalisé sur l’application qui affiche un schéma mathématique21.

Il n’y a pas de fichiers fournis.

TP 3 : La classe Cercle

Le TP 322 propose d’écrire une classe Cercle (y compris son programme de test et sa documentation !).

Plusieurs programmes de test sont fournis. SujetCercleTest permet de vérifier, voir compléter et préciser les exigences du sujet. RobustesseCercleTest réalise des tests de robustesse (vérification des assert en début de méthodes). FormeCercleTest vérification de quelques consignes et règles de programmation.

Voici les fichiers fournis23.

TP 4 : La classe Polygone

Le TP 424 propose d’écrire une classe Polygone (y compris son programme de test et sa documentation !). L’objectif est de manipuler les tableaux et les structures de contrôle de java.

Voici les fichiers fournis25.

CM 2 : Relations entre classes et diagrammes de séquence

Le cours26 présente les relations d’utilisation entre classes (association, aggrégation et composition) ainsi que le diagramme de séquence comme le moyen d’expliciter les interactions entre objets

CM 3 : Interfaces et généricité

Le cours27 présente les notions d’interface et de généricité.

TD 4 : Ensemble ordonné

Le TD 428 propose de modéliser un ensemble ordonné utilisé dans la résolution (naïve) du crible d’Ératosthène.

TP 5 : Afficher graphiquement les points et les segments

Le TP 529 propose de se familiariser avec un afficheur graphique dont la documentation30 est donnée puis de l’utiliser pour afficher graphiquement les points et les segments.

Sous Eclipse (ou équivalent), on utilisera le fichier d’archive afficheur.jar31 qu’il faudra ajouter comme « External archive file » au « Build path » du projet Java.

Voici les fichiers fournis32.
